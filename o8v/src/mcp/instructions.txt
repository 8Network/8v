8v — code reliability tool for AI agents. Use `8v` for all file operations (read, edit, write, search, inspect). Use shell tools only for git, process management, and environment operations. If an `8v` MCP tool is available, call it directly — do not shell out via Bash.

## Two principles, every command
1. **Progressive.** Default output is the minimum useful answer. Flags escalate detail (`read` returns a symbol map; `:start-end` for a range, `--full` for everything).
2. **Batch.** Pass many inputs in one call (`8v read a.rs b.rs c.rs`) instead of calling N times.

## Discovery
`8v ls --tree --loc` — full file hierarchy with line counts. Start here.
`8v ls [--match <glob>] [--stack <name>] [path]` — filtered views.
`8v search <pattern> [path]` — regex search across files.

## Read — symbol map first, range second, full last
`8v read <path>` — symbol map (line-number + symbol name). Use line numbers with `:start-end`.
`8v read <path>:<start>-<end>` — line range (1-indexed, end inclusive).
`8v read <path> --full` — entire file. Last resort.
`8v read a.rs b.rs Cargo.toml` — batch multiple files or ranges in one call.

## Write — prefer targeted edits
`8v write <path>:<line> "<content>"` — replace a single line.
`8v write <path>:<start>-<end> "<content>"` — replace a range.
`8v write <path>:<start>-<end> --delete` — delete a range.
`8v write <path>:<line> --insert "<content>"` — insert before a line.
`8v write <path> --find "<old>" --replace "<new>"` — fails if `<old>` not found.
`8v write <path> --append "<content>"`
Content args: `\n` = newline, `\t` = tab, `\\` = backslash. Pass as literal two-char sequences — do not rely on shell interpolation.

## Verify
`8v check .` — lint + type-check + format-check.
`8v fmt .` — auto-format in place.
`8v test .` — run tests.
`8v build .` — compile.
